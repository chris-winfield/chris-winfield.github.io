GitHub, Atom, and Markdown for Creative Writers

Chris Winfield

Introduction

The primary target of GitHub, a popular software collaboration and hosting platform, and Atom, a text editor developed by GitHub, is coders. Software engineers, web designers, game developers: coders. Most of the default features for GitHub and Atom make coding easier. For many of the same reasons these tools work so well for coding, creative writers benefit as well. While the initial set-up may appear daunting, contemporary computer users should face no issues. It will only take a few commits, or updates, to become fluent.

GitHub, with its ability to access files across multiple devices and version control, or revision tracking, forms the technological basis, while Atom serves as the writing implement. Atom is a text editor, and therefore will not always compare to more robust word processors, but its potential for customization provides the kind of fluidity so often necessary in the revision process.

This guide covers:

  • Registering for GitHub and creating your first repository
  • Installing GitHub Desktop and Atom
  • Committing for the first time
  • Customizing Atom for writing and using Atom effectively
  • Working with Markdown to create formatted documents
  • Exploring additional GitHub features

These initial steps will take around an hour to complete. They are intended for Windows and macOS users. When ready, begin with registration.
