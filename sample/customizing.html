GitHub, Atom, and Markdown for Creative Writers

Chris Winfield

Customizing Atom for Writing

With Atom now corresponding with GitHub all on its own, the focus now shifts to sharpening it as a writing implement. From the menu bar at the top of the screen, select "File" and then "Settings." The first category is "Core Settings." Most of these settings can be left alone for creative writing, though "Restore Previous Windows on Start" may be toggled off for a clean slate when starting a new session.

Under the "Editor" category, changing the font and font size, if desired, can take away some of the unpolished feeling of the text editor, as the default lives closer to a typewriter than to a modern word processor. Turning on "Soft Wrap" will force the editor to make paragraphs look like paragraphs instead of long lines that need to be scrolled across horizontally.

Now navigate to "Packages." Packages add functionality to Atom, similar to extensions on a web browser. Atom comes installed with dozens of packages, a good deal of them aimed towards coders. Many of these packages make writing in different programming languages easier by popping up suggestions in certain contexts. These may be disabled here if they prove annoying.

In "Themes," the appearance of Atom may be adjusted. The dilemma is between dark or light themes. The former generally lessens eye strain while the latter makes for a more familiar experience.

Select "Install" to install new packages. The Atom user community develops most of these. To begin, search "word count". The first two packages returned are "word-count" and "count-word." "Word-count" is deprecated, meaning that its development has not kept up with Atom, making it unusable. However, "count-word" appears correct. Select "Install" for "count-word" and Atom will download and install it in a few moments.

After that, install the following packages:

  • markdown-writer, which assists with Markdown, the formatting language used to make finished work more attractive
  • pandoc-convert, which converts the .md files to other formats
  • tool-bar
  • tool-bar-markdown-writer, which works with tool-bar and markdown-writer to make a friendly toolbar at the top of Atom

There are many other useful and optional packages for writers, including typewriter, zen, atom-beautify, and linter-write-good (a linter is a programming tool that identifies syntax errors and style problems in a language; this one serves that purpose for English).

Once everything installs, consider Atom fully outfitted for creative writing. Now begins the process of using it effectively. Close the settings tab and look at the interface for Atom. In this set-up process, it may be apparent that Atom splits itself into multiple panes, or groups of tabs. For example, selecting the GitHub or Git buttons in the lower right corner opens the GitHub/Git pane on the far right, which divides into two tabs, which work the same as tabs in a web browser, for those two functions. The left side always contains the Project pane, which displays the structure of the repository.

In the Project pane, open the file from the first commit and then make a new file and open it. The two files open in two tabs. Tabs can be transformed into panes to view two files side by side. Click and drag the second file's tab from the tab bar and move the cursor down to the pane. Notice the shadow effect. Releasing the mouse will create a new pane with that tab in the shadowed position, which can make a horizontal or vertical split. This provides an ideal arrangement for revision and editing work.

Click-dragging the last tab in a pane back to the tab bar of another pane eliminates the empty pane. Notice that tabs can also be rearranged within a pane this way. The size of panes can be adjusted by moving the mouse over the border line between two panes.

The repository folder view also allows for powerful manipulation. Files and folders can be renamed and moved. Double-clicking a file will open it permanently in a new tab, while a single click will open it in a tab until another file is selected. In a complex project with notes, character sketches, and an outline, the project tab becomes invaluable.

The different colors in the folder view highlight differences in the local repository from the committed version, which can be useful when writing comments at the end of a session.

The most powerful tool in Atom above these others is the command palette. The command palette gives quick access to any function in any installed package of Atom. Open the command palette by using control-shift-p (on Windows) or command-shift-p (on macOS) and type "count." The first option that appears is the toggle for the count-word tool. A toggled option means that its command must be pressed again to be turned off. The command palette also uses a toggle.

When in doubt, use the command palette. In the next section, Markdown will be used to turn written text into beautiful documents.
